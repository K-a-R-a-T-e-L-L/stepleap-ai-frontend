Как пользователь взаимодействует с продуктом по шагам:

Вход
Пользователь заходит из Telegram-бота по кнопке или диплинку.
На первом экране он сразу понимает, зачем продукт:
не “пройти тест”, а понять свой трек, попробовать себя в мини-задачах и увидеть подходящие вакансии.
Первый выбор
Сразу после входа пользователь выбирает один из двух режимов:
Быстрый старт — 3–5 минут.
Точно и глубоко — 15–25 минут.

Это важный UX-момент: не грузить длинной анкетой в лоб, а дать выбор по уровню вовлечения.

Короткий онбординг
Пользователь отвечает на несколько простых вопросов:
возраст,
образование,
цель: подработка / стажировка / первая работа,
предпочтения: люди / данные / техника,
команда / соло,
стабильность / динамика.

Задача этого этапа — быстро собрать стартовый профиль без ощущения экзамена.

Первая ценность почти сразу
После короткого ввода пользователь уже получает:
3–5 возможных направлений,
объяснение, почему они подходят,
первичную оценку сильных сторон,
первый рекомендуемый следующий шаг.

То есть пользователь не ждёт “финального результата” 20 минут, а сразу получает зацепку.

Паспорт навыков
Дальше он попадает в экран “Паспорт навыков”.
Там интерфейс отвечает на вопрос: “кто я сейчас”.

Что там видно:
интересы,
soft skills,
hard skills,
предпочтительная рабочая среда,
текущая цель,
первые доказательства навыков.

Это не просто профиль. Это уже полуготовая база для карьерного движения.

Карьерная карта
Следующий ключевой экран — “карьерное метро” или карта пути.

Что видит пользователь:
несколько карьерных веток,
станции роста,
быстрый выход в первую роль,
ветки усиления навыков,
ближайший понятный шаг.

Здесь магия в том, что пользователь перестаёт думать абстрактно:
не “мне вроде подходит маркетинг”,
а “сейчас я на этом уровне, вот ближайшая станция, вот через 2 недели я могу выйти на стажировку”.

Эпизоды
Потом пользователь идёт в “Эпизоды”.
Это главная дифференциация продукта.

Два типа:
Try-Out — примерить профессию.
Skill Sprint — быстро прокачать конкретный навык.

После эпизода интерфейс показывает:
что получилось,
что пока слабее,
как это изменило карьерную карту,
какие вакансии стали ближе.

То есть система не просто разговаривает, а двигает пользователя по пути.

Вакансии
Потом пользователь открывает экран вакансий.

Но вакансии здесь не “списком”.
Для каждой карточки показывается:
процент совпадения,
что уже совпадает по навыкам,
какой есть разрыв,
короткий план закрытия разрыва.

Это очень сильный UX, потому что вакансия перестаёт быть пугающей.
Пользователь видит не “я не подхожу”, а “мне не хватает вот этого, и это можно закрыть за 3 шага”.

Proofs / доказательства
Отдельный экран хранит артефакты:
мини-кейсы,
тексты,
разборы,
ответы,
рефлексии,
мини-резюме.

Это нужно, чтобы пользователь видел накопление результата.
И позже это можно использовать как карьерное портфолио внутри продукта.

Возврат через Telegram-бота
После выхода из Mini App пользователь не теряется.
Бот возвращает его обратно короткими действиями:
новые вакансии,
эпизод дня,
напоминание пройти спринт,
разбор вакансии, которую пользователь переслал,
быстрый вход в нужный экран Mini App.

То есть бот — это не альтернатива интерфейсу, а внешний триггер возврата.
