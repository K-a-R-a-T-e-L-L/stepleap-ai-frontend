Remote OK = источник вакансий
DeepSeek = мозг, который чистит, структурирует и персонализирует

Это как раз тот случай, где не нужно заставлять модель “самой искать вакансии в интернете”. У DeepSeek в API основной паттерн для внешних данных — это Tool Calls, то есть модель работает с внешними инструментами, которые даёшь ты. А Remote OK уже даёт открытый feed с вакансиями, так что здесь логичнее брать вакансии из feed, а DeepSeek использовать поверх них для нормализации и рекомендаций. (api-docs.deepseek.com
) (remoteok.com
)

1. Что ты строишь

Твоя система по сути должна работать так:

1. Берёшь вакансии из Remote OK
через https://remoteok.com/api. JSON feed с сайта редиректит именно туда. На главной Remote OK отдельно перечислены Remote Jobs API, RSS feed и JSON feed. (remoteok.com
) (remoteok.com
)

2. Сохраняешь сырой ответ
потому что потом удобно:

дебажить,
делать re-processing,
не терять исходный текст вакансии,
проверять, как модель что-то интерпретировала.

3. Прогоняешь вакансии через DeepSeek
модель:

убирает мусор,
приводит всё к единой структуре,
вытаскивает навыки,
определяет seniority,
чистит description,
строит summary.

4. Потом DeepSeek или твоя логика матчит вакансии под пользователя
с учётом:

интересов,
уровня,
цели,
формата работы,
навыков,
skill gap.
2. Как подключить DeepSeek

DeepSeek API использует формат, совместимый с OpenAI SDK. В их документации указано, что можно использовать OpenAI SDK, если задать base_url="https://api.deepseek.com", а авторизация идёт через Bearer API key. Для обычных задач у них используются модели deepseek-chat и deepseek-reasoner. (api-docs.deepseek.com
) (api-docs.deepseek.com
)

Минимально это выглядит так:

import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: "https://api.deepseek.com",
});

Для твоего кейса чаще всего хватит deepseek-chat.
deepseek-reasoner лучше оставить для более сложной логики, если захочешь делать глубокий reasoning по карьерному пути. (api-docs.deepseek.com
)

3. Как подключить Remote OK

Remote OK лучше использовать как открытый внешний feed, а не как HTML-парсинг.

Запрос:
curl https://remoteok.com/api

Ответ — это JSON-массив.
Первый элемент массива — служебный, не вакансия. Там лежат last_updated и legal. Дальше уже идут реальные вакансии с полями вроде:

id
date
company
position
tags
description
location
apply_url
salary_min
salary_max
url (remoteok.com
)

Очень важно: в legal Remote OK пишет, что нужно ссылаться обратно на URL вакансии и упоминать Remote OK как источник, иначе доступ могут ограничить. (remoteok.com
)

6. Какой пайплайн делать
Этап A — получение вакансий
cron раз в 10–30 минут делает запрос к Remote OK API
получает JSON
пропускает первый элемент массива
сохраняет raw-вакансии
Пример:

const data = await res.json();

Это соответствует структуре JSON feed Remote OK. (remoteok.com
)

Этап B — нормализация через DeepSeek

Берёшь одну вакансию и отправляешь её в DeepSeek.

Вот базовый промпт для нормализации вакансии.

SYSTEM PROMPT — vacancy normalizer
Ты — AI-нормализатор вакансий для карьерного сервиса.

Твоя задача:
1. Получить сырую вакансию из job feed.
2. Убрать мусор, маркетинговую воду, спам-фразы, повторяющиеся блоки, мусорные хештеги и лишние CTA.
3. Вытащить структурированные поля.
4. Ничего не выдумывать. Если данных нет — ставь null или пустой массив.
5. Отвечай строго JSON-объектом без markdown и без комментариев.

Правила:
- Не придумывай компанию, зарплату, location, если их нет.
- Не дублируй сырое описание.
- Описание делай коротким, чистым и читаемым.
- Навыки выводи в виде массива строк.
- Если seniority неочевиден — ставь "unknown".
- Если формат работы неочевиден — ставь "unknown".
- Если вакансия похожа на спам, MLM, мутную крипто-схему, фальшивый рекрутинг или явный low-trust пост — поставь low_quality=true.
- Если вакансия релевантна новичкам, стажёрам или junior-кандидатам — отметь junior_friendly=true.
- Не используй внешний контекст.
USER PROMPT — vacancy normalizer
Нормализуй вакансию в JSON по следующей схеме:

{

8. Как делать персональный матчинг

После того как у тебя есть чистые вакансии, следующий шаг — матчит их под пользователя.

Тут уже нужен второй промпт.

SYSTEM PROMPT — vacancy matcher
Ты — AI-карьерный ассистент.

Твоя задача:
1. Получить профиль пользователя и одну нормализованную вакансию.
2. Оценить, насколько вакансия подходит пользователю.
3. Объяснить это простым языком.
4. Показать skill gap.
5. Сформулировать конкретные next steps.

Правила:
- Не будь слишком оптимистичным.
- Не ставь высокий match, если человеку явно не хватает базовых требований.
- Не выдумывай опыт или навыки пользователя.
- Оценивай по интересам, формату работы, навыкам, seniority и цели пользователя.
- Отвечай строго JSON.
USER PROMPT — vacancy matcher
Оцени матч между пользователем и вакансией.

Верни JSON по схеме:

{
  "match_score": 0,
  "fit_level": "low|medium|high",
  "why_fit": ["string"],
  "why_not_fit": ["string"],
  "skill_gap": ["string"],
  "next_steps": ["string"],
  "recommended": true
}

Профиль пользователя:
{{USER_PROFILE_JSON}}

Вакансия:
{{NORMALIZED_VACANCY_JSON}}

13. Самый короткий рабочий старт

Если тебе нужно стартовать максимально быстро, делай так:

Шаг 1

Подключи DeepSeek через OpenAI SDK-compatible клиент с base_url=https://api.deepseek.com. (api-docs.deepseek.com
)

Шаг 2

Подключи Remote OK через https://remoteok.com/api. (remoteok.com
)

Шаг 3

Игнорируй первый элемент массива.

Шаг 4

Прогоняй каждую вакансию через vacancy normalizer prompt.

Шаг 5

Сохраняй нормализованные вакансии.

Шаг 6

Когда пользователь открывает Mini App, прогоняй его профиль и пачку вакансий через vacancy matcher prompt.

Шаг 7

Выводи лучшие 10–20 вакансий с объяснением “почему”.
